# 기능명세서 VER 0.1

## 서비스 개요

- 기능명: 하루 요정
- 기능 한 줄 설명: AI와 대화하면, 그 대화를 바탕으로 짧은 메모와 해야 할 체크리스트를 자동으로 정리해 준다.
- 핵심 가설: 백지에 직접 쓰는 대신 AI와 짧게 대화해 일기가 자동 완성되면, 시간 부담이 낮아져 사용자가 기록을 꾸준히 이어갈 것이다.
- 선정 이유:
  - 다이어리 작성 전 사용자 설문에서 "매일 기록할 시간을 내기 어려움"이 가장 큰 페인포인트로 확인됨.
  - 응답자의 66.7%가 기록에 쓸 수 있는 시간이 10분 이내라고 답함.
  - 따라서 사용자의 기록 부담을 낮추는 핵심 기능을 AI 채팅 요약으로 정의함.

## User Flow

1. 사용자가 AI 채팅 화면에서 오늘 있었던 일, 감정, 내일 해야 할 일을 AI와 대화로 입력한다.
2. AI가 대화 내용을 수집하고 오늘의 기록, 내일 할 일, 일정 후보를 구조화한다.
3. 사용자가 "대화 마치고 정리하기" 버튼을 누르면 AI 정리 결과 화면으로 이동한다.
4. AI 정리 결과 화면에서 감정일기, 할 일, 일정 후보를 카드 형태로 확인한다.
5. 필요하면 메모 수정 또는 할 일 수정 화면에서 내용을 수정한다.
6. 일정 후보가 있으면 등록 또는 무시를 선택한다.
7. 사용자가 "저장하고 완료" 버튼을 누르면 기록, 할 일, 일정이 저장된다.
8. 홈 화면에서 전날 AI 대화로 생성된 오늘의 할 일과 일정을 확인한다.
9. 캘린더 화면에서 선택 날짜의 일정 목록과 시간 정보를 확인한다.
10. 홈 화면 또는 기록 탭에서 할 일을 완료 처리하거나 메모/할 일을 직접 추가한다.
11. 사용자가 다시 AI와 대화하며 루틴을 반복한다.

## 화면별 기능 상세

### 1. 홈 화면

#### 헤더

- 사용자 인사 영역: 프로필 정보의 사용자 이름을 불러와 "안녕하세요, {사용자이름}님"을 표시한다.
- 오늘 날짜 표시: 기기 기준 날짜를 "6월 30일 화요일" 형식으로 표시한다.
- 연속 기록 뱃지: 기록 작성 이력 기준으로 연속 기록 일수를 계산해 "🔥 5일째" 형태로 표시한다. 하루 미기록 시 0일째로 초기화한다.

#### AI 대화 진입 영역

- AI 대화 진입 카드: 사용자가 AI와 대화를 시작하도록 유도한다.
- 안내 문구:
  - "기록하고 싶은 내용이 있나요?"는 볼드 처리한다.
  - "AI요정 하루가 정리해드릴게요"는 기본 서체로 표시한다.
- AI 대화 아이콘 버튼: 카드 우측 말풍선 아이콘 클릭 시 AI 채팅 화면으로 이동한다.
- PC hover: 컴포넌트 채도를 살짝 올리고 커서를 손모양으로 변경한다.

#### 캘린더 요약 영역

- 미니 캘린더: 현재 월 날짜를 축약 캘린더로 표시하고 오늘 날짜를 원형 강조한다.
- 캘린더 클릭 시 캘린더 탭으로 이동한다.
- 당일 날짜를 크게 표시한다.
- 당일 일정 목록: 오늘 일정이 있으면 일정명과 시간을 표시한다. 없으면 "등록된 일정이 없습니다"를 표시한다.

#### 오늘의 할 일 카드

- 오늘의 할 일 목록을 체크리스트 형태로 표시한다.
- 카드 영역 클릭 시 기록 화면의 할 일 탭으로 이동한다.
- 오늘 할 일이 없으면 "등록된 일정이 없습니다"를 표시한다.
- 완료 현황을 "1/4 완료" 형태로 표시한다.
- 체크박스 클릭 시 완료 처리하고 텍스트에 취소선을 적용한다. 재클릭 시 원복한다.
- "할 일 추가" 버튼 클릭 시 맨 아래 빈 항목을 생성한다.
- 작성하지 않은 빈 항목은 자동 삭제한다.

#### 하단 네비게이션 바

- 탭: 홈, 캘린더, AI 채팅, 기록, 마이
- 각 탭은 버튼 형식이며 클릭 시 해당 화면으로 이동한다.
- 선택된 탭은 활성화 색상을 적용한다.
- PC hover: 컴포넌트 채도를 살짝 올리고 커서를 손모양으로 변경한다.

### 2. 캘린더 화면

#### 월간 캘린더

- 하단 탭바의 캘린더 탭 또는 홈 미니 캘린더 클릭 시 진입한다.
- 현재 보고 있는 연월을 "2026년 6월" 형식으로 표시한다.
- 좌우 화살표 버튼으로 이전 월, 다음 월로 이동한다.
- 일~토 요일을 표시하고 토요일은 파란색, 일요일은 빨간색으로 강조한다.
- 날짜를 월간 그리드로 표시한다.
- 오늘 날짜는 full-fill 컬러 원형으로 표시한다.
- 선택 날짜는 테두리 원형으로 표시한다.
- 일정이 있는 날짜는 날짜 아래 작은 점을 표시한다. 최대 3개까지 시간순으로 표시한다.

#### 선택 날짜 일정

- 선택한 날짜와 일정 개수를 "6월 29일 일요일 · 일정 1개" 형태로 표시한다.
- 일정 카드 목록은 시간순 정렬한다.
- 일정이 없으면 "등록된 일정이 없습니다"를 표시한다.
- 일정 카드에는 시작 시간, 일정명, 카테고리 색상 바를 표시한다.
- 일정 카드 클릭 시 상세/수정/삭제 화면으로 이동한다.
- 우측 하단 + 플로팅 버튼 클릭 시 새 일정 작성 팝업을 연다.

### 2-1. 캘린더 작성 팝업

- 새 일정을 등록할 수 있는 팝업을 제공한다.
- 팝업은 네비게이션 바를 덮는 형태로 노출한다.
- 외부 영역 클릭으로는 닫지 않고 X 버튼으로만 닫는다.
- X 버튼 클릭 시 "작성을 취소하시겠습니까? 변경한 내용은 저장되지 않아요." 확인 팝업을 표시한다.
- 입력 필수값: 제목, 색상
- 제목은 최대 30자 미만으로 제한한다.
- 시작일과 종료일을 설정할 수 있고, 캘린더 선택 시 요일을 자동 반영한다.
- 종료일이 시작일보다 이전이면 저장을 제한한다.
- 반복 요일은 일~토 버튼으로 복수 선택 가능하며 선택 시 full-fill 컬러로 표시한다.
- 시간 설정 토글 기본값은 OFF이며, OFF일 때 하루 종일 일정으로 등록한다.
- 토글 ON 시 시작 시간과 종료 시간 입력 필드를 표시한다.
- 시간 설정 ON 기본값은 시작 09:00, 종료 10:00이다.
- 시작 시간 변경 시 종료 시간이 시작 시간 이후가 되도록 자동 보정한다.
- 종료 시간이 시작 시간보다 빠르거나 같으면 저장을 제한하고 "종료 시간은 시작 시간 이후여야 합니다."를 표시한다.
- 색상은 1개만 선택할 수 있다.
- "일정 추가" 버튼은 필수값 충족 시 활성화하며 저장 후 팝업을 닫고 캘린더에 반영한다.

### 3. AI 채팅 화면 - 채팅중

#### 헤더

- 화면 제목: "대화"
- 안내 문구: "편하게 대화하시면 메모, 일정 등을 AI가 정리해드려요"

#### 채팅 영역

- 사용자 메시지는 우측 정렬 말풍선으로 표시한다.
- AI 메시지는 좌측 정렬 말풍선과 나비 아이콘으로 표시한다.
- 대화 누적 시 스크롤로 확인한다.
- AI는 대화 내용을 바탕으로 일정과 할 일을 추출한 카드도 표시한다.
- 추출 카드 예시:
  - "📆 일정 · 치과 6/30 15:00"
  - "✅ 할 일 · 보고서 초안"

#### 입력 영역

- 메시지 입력창 플레이스홀더: "답장을 입력하세요"
- 메시지 전송 버튼은 한 글자 이상 입력 시 활성화한다.
- 버튼은 화살표 아이콘 형태로 표시한다.
- 키보드 엔터로는 전송하지 않는다.
- "대화 마치고 정리하기" 버튼 클릭 시 AI 정리 결과 화면으로 이동한다.

### 3-1. AI 채팅 화면 - 채팅 종료 결과

#### 헤더

- 화면 제목: "오늘의 기록이 완성됐어요 ✨"
- 안내 문구: "AI가 대화를 바탕으로 작성했어요. 확인하고 저장해주세요."

#### 메모 컴포넌트

- AI가 생성한 제목과 본문을 표시한다.
- 예시 제목: "바쁘지만 알차던 하루"
- 예시 본문: "오전엔 면접 준비로..."
- 수정 버튼 클릭 시 메모 수정 팝업을 연다.

#### To-do List 컴포넌트

- AI가 생성한 할 일 체크리스트를 표시한다.
- 결과 화면에서는 체크박스 완료 처리를 하지 않는다.
- 수정 버튼 클릭 시 할 일 수정 팝업을 연다.

#### 일정 등록 제안 컴포넌트

- 날짜와 시간이 포함된 내용을 감지하면 일정 등록 제안 카드를 표시한다.
- 일정명, 날짜, 시간 정보를 표시한다.
- 예시: "치과 예약 / 6/30(화) 오후 3:00"
- 수정 버튼 클릭 시 일정 수정 팝업을 연다.
- 무시 버튼 클릭 시 제안 카드를 숨긴다.
- 등록 버튼 클릭 시 캘린더에 해당 일정을 생성한다.

#### 저장하기 컴포넌트

- "저장하고 완료" 버튼 클릭 시 저장 완료 팝업을 표시한다.
- 저장 완료 시:
  - 메모는 기록 화면의 메모 탭에 반영한다.
  - 할 일은 기록 화면의 할 일 탭에 반영한다.
  - 일정은 홈 캘린더 컴포넌트와 캘린더 화면에 반영한다.

### 3-2. AI 채팅 종료 후 - 메모 수정

- 취소 버튼 클릭 시 "작성을 종료하시겠습니까? 변경한 내용은 저장되지 않아요." 확인 팝업을 표시한다.
- 저장 버튼은 본문 1자 이상 작성 시 활성화한다.
- 날짜를 확인 및 수정할 수 있다.
- 제목과 본문은 AI 생성값을 기본값으로 노출하고 직접 수정 가능하다.
- 제목 또는 본문이 공란이면 저장을 제한한다.
- 저장 시 수정 팝업을 닫고 AI 정리 결과 화면으로 돌아간다.

### 3-3. AI 채팅 종료 후 - 할 일 수정

- 취소 버튼 클릭 시 작성 종료 확인 팝업을 표시한다.
- 저장 버튼 클릭 시 내용 저장 후 AI 정리 결과 화면으로 돌아간다.
- 빈 항목은 제외하고 저장한다.
- 날짜를 확인 및 수정할 수 있다.
- 체크박스는 모양만 구현하고 클릭은 불가하다.
- 각 할 일 항목은 텍스트 박스에서 바로 수정할 수 있다.
- 각 항목 우측 X 버튼으로 항목을 삭제할 수 있다.
- "할 일 추가" 버튼 클릭 시 맨 아래 빈 항목을 생성한다.

### 3-4. AI 채팅 종료 후 - 일정 수정

- 취소 버튼 클릭 시 작성 종료 확인 팝업을 표시한다.
- 저장 버튼 클릭 시 내용 저장 후 AI 정리 결과 화면으로 돌아간다.
- 일정 이름은 텍스트 박스로 수정한다.
- 일정 이름은 최대 20자까지 입력한다.
- 공란으로 저장하면 기존 AI 생성 이름을 유지한다.
- 날짜와 시간은 버튼 영역 클릭 시 달력/시간 컴포넌트를 통해 수정한다.

### 3-5. AI 채팅 종료 후 - 저장 완료

- 저장 완료 팝업 문구:
  - "저장 완료"
  - "오늘의 기록이 안전하게 저장되었어요."
- 체크 아이콘과 저장 완료 문구를 표시한다.
- 확인 버튼 클릭 시 기록 화면으로 이동한다.

### 4. 기록 화면 - 메모 탭

- 화면 제목: "기록"
- 기록 유형은 메모, 할 일 토글로 분리한다.
- 기록 화면 진입 시 기본값은 메모 탭이다.
- 저장된 기록은 날짜 기준으로 그룹핑한다.
- 한 날짜에 여러 메모가 있으면 최신순으로 정렬한다.
- 메모 카드는 제목과 본문 요약으로 구성한다.
- 카드 클릭 시 작성된 메모를 팝업으로 표시한다.
- 더보기 버튼 클릭 시 메모 본문 페이지로 이동한다.
- 메모 본문 페이지는 제목, 날짜, 본문으로 구성한다.
- 목록으로 돌아가기 버튼으로 이전 화면으로 이동한다.
- 메모 수정 버튼 클릭 시 수정 화면으로 이동한다.
- 플로팅 작성 버튼 클릭 시 메모 작성 화면으로 이동한다.

### 4-1. 기록 화면 - 메모 작성창

- 작성창 토글로 메모와 할 일을 전환할 수 있다.
- 메모 화면에서 작성 버튼을 누르면 메모 토글이 기본 활성화된다.
- 취소 버튼 클릭 시 작성 내용이 있을 경우 "작성을 취소하시겠습니까? 변경한 내용은 저장되지 않아요." 팝업을 표시한다.
- 저장 버튼은 본문 1자 이상 작성 시 활성화한다.
- 날짜는 기본값을 당일로 두고 수정할 수 있다.
- 제목을 작성하지 않으면 본문 최초 10글자를 제목으로 설정한다.
- 본문 플레이스홀더: "내용을 입력하세요."

### 4. 기록 화면 - 할 일 탭

- 기록 유형 토글에서 할 일을 선택하면 할 일 목록을 표시한다.
- 저장된 할 일은 날짜별로 그룹핑하고 최신 날짜순으로 정렬한다.
- 같은 날짜의 기존 체크리스트가 있으면 새 카드 생성이 아니라 기존 카드에 추가한다.
- 날짜별 완료 현황을 "1/4 완료" 형태로 표시한다.
- 할 일 항목 클릭 시 체크박스 활성화와 취소선을 적용한다. 재클릭 시 원복한다.
- 수정 버튼 클릭 시 내용이 채워진 할 일 작성/수정 화면으로 이동한다.
- 플로팅 작성 버튼 클릭 시 비어 있는 할 일 작성 화면으로 이동한다.

### 4-2. 기록 화면 - 할 일 작성창

- 작성창 토글로 메모와 할 일을 전환할 수 있다.
- 할 일 화면에서 작성 버튼을 누르면 할 일 토글이 기본 활성화된다.
- 취소 버튼 클릭 시 작성 종료 확인 팝업을 표시한다.
- 저장 버튼은 사용자가 1글자 이상 작성 시 활성화한다.
- 날짜는 기본값을 당일로 두고 수정할 수 있다.
- 할 일 입력 플레이스홀더: "할 일을 입력하세요."
- 엔터 입력 시 다음 칸을 생성하고 텍스트 커서를 이동한다.
- X 버튼 클릭 시 해당 줄을 삭제한다.
- "할 일 추가" 버튼 클릭 시 다음 칸을 생성하고 텍스트 커서를 이동한다.

### 5. 마이 탭 - 로그인 전

- 화면 제목: "마이"
- 비로그인 사용자에게만 로그인 안내 카드를 표시한다.
- 기본 사용자 프로필 아이콘을 표시한다.
- 로그인 유도 문구: "로그인하고 기록을 지켜요"
- 보조 문구: "여러 기기에서 동기화하고 안전하게 백업해요"
- SNS 로그인 버튼을 제공한다. 개발 단계에서는 구현 편의성이 높은 SNS를 선택한다.
- 회원가입 버튼 클릭 시 회원가입 페이지로 이동한다.
- 하단 네비게이션 바는 홈, 캘린더, AI 채팅, 기록, 마이를 표시한다.
- AI 채팅 중앙 버튼은 다른 탭보다 강조한다.

### 5-1. 마이 탭 - 회원가입 페이지

- 돌아가기 버튼 클릭 시 작성 종료 확인 팝업을 표시한다.
- 상단 중앙에 "회원가입" 제목을 표시한다.
- 안내 문구:
  - "하루 요정 시작하기"
  - "닉네임과 비밀번호로 계정을 만들어요"
- 자체 회원가입 정보: 닉네임, 아이디, 비밀번호
- 아이디는 중복 금지이며 중복 시 입력 박스를 빨간색으로 변경하고 안내 문구를 노출한다.
- 아이디 플레이스홀더: "사용할 아이디"
- 닉네임은 중복 가능하며 한글, 영어, 숫자 제한은 없다.
- 닉네임 플레이스홀더: "사용할 닉네임"
- 비밀번호는 숫자 4자리만 가능하다.
- 비밀번호 규정이 맞지 않으면 에러 스타일과 안내 문구를 노출한다.
- 비밀번호 플레이스홀더: "비밀번호 입력"
- 비밀번호 확인 버튼으로 입력값 노출/비노출을 토글한다.
- 가입하기 버튼 클릭 시 회원가입 완료 팝업을 표시한다.

## 예외 처리

### 빈 화면 / 데이터 없음

- 저장된 일정, 기록, 할 일이 없을 경우 빈 화면 안내 문구를 표시한다.
- 예시:
  - "아직 등록된 기록이 없어요."
  - "오늘 일정이 없어요."

### 로딩 상태

- AI 응답 생성, 저장, 로그인, 캘린더 반영 등 시간이 필요한 처리에는 로딩 상태를 표시한다.
- 예시:
  - "하루를 정리하는 중이에요..."
  - "저장 중이에요..."

### 에러 상태

- 네트워크 오류, 저장 실패, AI 응답 실패 등 문제가 발생하면 에러 안내 문구와 재시도 버튼을 제공한다.
- 예시: "일시적인 오류가 발생했어요. 다시 시도해주세요."

### 입력값 누락

- 필수 입력값이 비어 있으면 저장 또는 등록 버튼을 비활성화한다.
- 대상 예시: 일정 제목, 메모 내용

### 글자 수 초과

- 제목, 메모, 할 일 입력값이 제한 글자 수를 초과하면 더 이상 입력되지 않도록 하거나 초과 안내 문구를 표시한다.
- 예시: "제목은 최대 30자까지 입력할 수 있어요."

### 중복 입력

- 동일한 시간의 일정이나 동일한 할 일이 중복 입력될 경우 사용자에게 확인 메시지를 표시한다.
- 예시:
  - "같은시간대에 일정이 이미 있어요. 일정을 바꿀까요?"
  - "이미 동일한 할 일이 입력되어 이 이름은 사용할 수 없어요."

### 날짜/시간 오류

- 종료일이 시작일보다 빠르거나 종료 시간이 시작 시간보다 빠른 경우 저장을 제한하고 안내 문구를 표시한다.
- 예시: "종료 시간은 시작 시간 이후로 설정해주세요."

### AI 응답 실패

- AI가 응답을 생성하지 못하면 재시도 버튼을 제공한다.
- 예시: "AI 정리에 실패했어요. 다시 시도해주세요."

### 저장 전 이탈

- 작성 또는 수정 중 화면을 벗어나려 하면 확인 팝업을 표시한다.
- 예시: "작성 중인 내용이 저장되지 않을 수 있어요. 나가시겠어요?"

### 삭제 동작

- 메모, 할 일, 일정 삭제 시 즉시 삭제하지 않고 확인 팝업을 표시한다.
- 예시: "정말 삭제하시겠어요?"

### 반복 일정 저장

- 반복 요일이 선택되면 해당 규칙에 맞는 반복 일정으로 저장한다.
- 반복 일정도 캘린더 점 표시 규칙을 따른다.
- 점 표시 순서는 시간순이며, 시간 지정이 없는 종일 일정은 후순위로 둔다.

### 팝업 외부 영역 처리

- 일정 작성 팝업은 외부 클릭으로 닫지 않는다.
- 다른 팝업의 외부 클릭 닫기 여부는 화면별 UX 정책에 맞춰 구현한다.

## 구현 메모

- 신규 앱으로 구현한다. `team-final-5`의 취미 추천 서비스 코드와 도메인 로직은 사용하지 않는다.
- MVP 우선순위는 AI 채팅으로 메모, 할 일, 일정 후보를 생성하고 저장하는 흐름이다.
- 실제 AI 연동 전에는 로컬 mock 응답으로 핵심 플로우를 먼저 구현할 수 있다.
- 로컬 저장소 또는 클라이언트 상태로 MVP를 구현한 뒤 로그인/동기화는 후순위로 확장한다.
